Several problems

Some general problems that might occur.

  • Bots sometimes seem to float instead of walking. This is a result of the Quake2 player animation code when the bot/player is just a few inches from the ground.
  • Sometimes bots just stand still and do nothing. The bot can't leave it's current area due to a reachability calculation problem.
  • The bot doesn't know how to go about with buttons, shootable doors, secret doors, some platforms, trains, rotating objects, lasters, triggers, bridges, traps etc. As a result the bot might sometimes do stupid things or simply get stuck on maps with lots of these features. The bot currently understands a lot of these structural systems but simply won't perform any actions to use them.
  • The bot sometimes has troubles climbing a ladder. This is a problem with the Quake2 physics code. If a ladder brush and a solid brush overlap and have sides in the same plane then the Quake2 ladder physics code seems to be unpredictable. The plane might work as a ladder but there seems to be no way to tell for sure.
